## xml과 json 형식의 차이

http://tcpschool.com/json/json_intro_xml

차이점
1. JSON은 종료 태그를 사용하지 않습니다.

2. JSON의 구문이 XML의 구문보다 더 짧습니다.

3. JSON 데이터가 XML 데이터보다 더 빨리 읽고 쓸 수 있습니다.

4. XML은 배열을 사용할 수 없지만, JSON은 배열을 사용할 수 있습니다.

5. XML은 XML 파서로 파싱되며, JSON은 자바스크립트 표준 함수인 eval() 함수로 파싱됩니다.

# 동네예보정보조회서비스 
-> 컨트롤러로 타면 안됨 because main index 페이지 뜨는 시간이 중요한데, 이미지 및 각종쿼리등을 수행하는 시간이 3s(초)가 넘으면 안되므로, 
메인페이지에서 가장 중요한것중 하나는 페이지 로딩시간임. 따라서 비동기로 처리하는 방식이 적합

-> 동네예보정보조회서비스는 트래픽이 1000이므로 1천이 넘어가면 에러가 발생할 수 있음. 따라서 DB나 파일로 떨군 후에, 해당 정보를 조회해서 뿌리는 방식을 취해야함
-> 메인페이지에 띄우는거면 DB의 경우 쿼리문이 최소 1개이상 생기므로, 파일로떨구는게 적합할 수 도 있음
-> 파일로 떨굴때 어떤 형식으로 떨굴지 xml, json 
-> json으로 떨구는 경우 webapp 밑에 json폴더를 생성
-> 파일명은 weather.json으로 일치
-> 동네예보정보조회서비스 json에 들어가는 기본적인 key값 외에 추가적으로 updateTime이라는 속성을 VO에 추가 
-> 그 후 parse 메서드 호출 -> Object(vo)리턴 -> object to json 메서드 호출 -> json 형식의 String 데이터를 파라미터로 갖는 json 파일 생성 메서드 호출
-> json파일이 weather.jsp일 경우 해당 폴더/파일명을 url에 입력해서 json 값을 가져올 수 있음
-> 이에 대한 처리는 클라이언트 쪽에서 처리 
-> 그리고 정해진 시간마다 동작하게 스케줄러 등록

특징 : json 변환을 위해 ObjectMapper 사용, 동네예보정보조회서비스는 xml과 json 형식을 둘 다 지원
